这次作业围绕 CIFAR-10 图像分类展开，要求使用 PyTorch 自定义卷积神经网络，在不使用预训练模型的前提下完成数据加载、模型定义、训练循环和测试评估，并在 100 个 epoch 内让测试集准确率超过 80%。

完成这次作业之后，我对一个典型 PyTorch 训练任务的结构有了更清晰的认识：先准备训练集和测试集，再定义模型结构、损失函数和优化器；训练时通过前向传播得到预测结果，通过损失函数衡量预测误差，再通过反向传播计算梯度，并由优化器更新模型参数。

这篇记录主要保存两类内容：一是这次 CNN 作业的实现和训练结果，二是围绕训练过程产生的几个基础问题，包括模型参数存在哪里、CNN 通常由哪些层组成、PyTorch 训练脚本的基本结构、AdamW 和 BatchNorm 分别起什么作用。后续阅读代码时，又补充记录了常量、随机种子、卷积块参数、特征图和池化之间的关系。

## 源码结构

本次作业的完整源码保存在：

- `D:\projects\ai_homework\homework_02_cifar10_cnn\cifar10_cnn.py`

这份代码可以按训练任务的流程分成几个部分来看。

首先是环境和基础配置。代码开头导入 PyTorch、数据加载、神经网络模块等依赖，并定义 CIFAR-10 数据集的均值和标准差。均值和标准差用于图像标准化，使输入数据的数值分布更稳定。

接着是设备选择和随机种子设置。`set_seed` 用来固定随机性，使实验更容易复现；`get_device` 用来选择训练设备，优先使用可用的 MLU 或 CUDA GPU，否则回退到 CPU。

然后是数据预处理和数据加载。`build_transforms` 定义训练集和测试集的图像处理流程：训练集使用随机裁剪、水平翻转、颜色扰动和随机擦除等数据增强方法，测试集只做张量转换和标准化。`build_dataloaders` 使用 `torchvision.datasets.CIFAR10` 读取数据集，并通过 `DataLoader` 构造训练和测试批次。

模型结构由 `ConvBlock` 和 `Cifar10CNN` 两部分组成。`ConvBlock` 是一个重复使用的卷积功能块，内部包含卷积层、BatchNorm 和 ReLU；`Cifar10CNN` 则把多个卷积功能块、池化层和最终分类层组合成完整的 CNN。输入图像经过卷积部分提取特征，最后通过线性层输出 10 个类别的预测结果。

训练逻辑主要在 `train_one_epoch` 中。每个 batch 会先经过前向传播得到预测结果，再用损失函数计算预测和真实标签之间的差距。随后调用 `loss.backward()` 计算梯度，并由优化器执行 `optimizer.step()` 更新模型参数。

测试逻辑在 `evaluate` 中。测试阶段会切换到 `model.eval()`，并使用 `torch.no_grad()` 关闭梯度计算，只评估模型在测试集上的 loss 和准确率。

最后，`main` 函数把这些部分串起来：解析命令行参数，构造数据集和模型，定义损失函数、优化器和学习率调度器，然后循环执行训练与测试，并把每个 epoch 的结果输出到日志中。

## 读代码时补充理解的问题

### 开头的常量有什么用？

代码开头的 `CIFAR10_MEAN` 和 `CIFAR10_STD` 不是模型训练出来的参数，而是 CIFAR-10 数据集 RGB 三个通道的均值和标准差。它们会被传给 `transforms.Normalize`，用于把图像张量做标准化：

- 先用 `ToTensor()` 把像素值转换到 `[0, 1]` 附近。
- 再用 `(像素值 - 均值) / 标准差` 调整每个颜色通道的分布。

这样做的目的不是直接增加模型容量，而是让输入数据的数值尺度更稳定。对于深一点的 CNN，标准化通常能让训练更稳定，也能降低学习率选择的难度。

### 固定随机种子是为了什么？

`set_seed` 的作用是尽量固定实验中的随机性。训练过程里有很多随机来源，例如模型初始权重、训练集打乱顺序、随机裁剪、随机水平翻转、随机擦除和 Dropout。

固定随机种子不等于提高模型能力，它主要是为了让实验结果更可比较。如果后续只改一个函数或一个超参数，固定随机性可以减少“这次准确率变化只是随机波动”的干扰。

### 什么叫“提取特征”？

“提取特征”可以理解为：模型把原始像素逐步转换成更适合分类的表示。

原始输入只是在描述每个位置的 RGB 数值；经过卷积层之后，中间特征图可能逐渐对应边缘、颜色块、纹理、局部形状，再到更接近类别判断的抽象模式。这些模式不是人工写死的，而是卷积核参数在训练中学出来的。

所以，卷积层不是直接回答“这是不是飞机/汽车/鸟”，而是先把图像变成一组中间表示，最后分类层再根据这些表示输出 10 个类别分数。

### `ConvBlock(in_channels, out_channels, dropout)` 里的参数是什么意思？

`in_channels` 表示输入特征图数量。第一层输入是 RGB 图像，所以是 3；后面的层接收上一层输出的特征图，所以可能是 64、128 或 256。

`out_channels` 表示当前卷积块要输出多少张新的特征图。可以把它理解为当前层要学习多少种新的特征探测方式。

`dropout` 表示训练时随机丢弃一部分通道的比例，用来减少模型对某些局部特征的过度依赖，从而缓解过拟合。

例如 `ConvBlock(64, 128, dropout=0.10)` 可以理解为：上一阶段已经产生 64 张特征图，当前阶段要在这些旧特征基础上组合出 128 张新的特征图，并在训练时加入少量随机丢弃。

### 一个输出通道和卷积核是什么关系？

一开始容易误以为“一个通道就是一个 3x3 卷积核”。更准确的理解是：

- 一个输出通道对应一组卷积核参数。
- 这一组参数会覆盖所有输入通道。
- 如果输入有 `in_channels` 张特征图，那么一个输出通道就需要 `in_channels` 个 `3x3` 小卷积核。
- 如果要输出 `out_channels` 张特征图，总共就需要 `in_channels * out_channels` 个 `3x3` 小卷积核。

例如 `nn.Conv2d(64, 128, kernel_size=3, padding=1, bias=False)` 中，输入是 64 张特征图，输出是 128 张特征图。为了生成每一张输出特征图，模型都需要综合全部 64 张输入特征图。因此小卷积核数量是：

```text
64 * 128 = 8192
```

如果只计算卷积权重参数数量，还要再乘以每个小卷积核的 `3 * 3`：

```text
64 * 128 * 3 * 3 = 73728
```

这也解释了为什么后面的卷积层参数会迅速变多：输入特征图数量和输出特征图数量都增加了。

### 卷积和池化分别怎样改变特征图？

在这次作业的模型里，普通卷积主要改变通道数量，不改变特征图宽高。原因是卷积层使用了 `kernel_size=3`、`stride=1` 和 `padding=1`。这种设置会让输出宽高和输入宽高保持一致。

例如：

```text
[batch, 3, 32, 32]
Conv2d(3, 64, kernel_size=3, padding=1)
[batch, 64, 32, 32]
```

宽高仍然是 `32x32`，但通道数从 3 变成 64。

池化层则主要负责缩小宽高。`MaxPool2d(kernel_size=2)` 默认步长为 2，所以会把宽高减半，但不改变通道数：

```text
[batch, 64, 32, 32]
MaxPool2d(2)
[batch, 64, 16, 16]
```

因此，这次模型的整体变化可以概括为：

```text
卷积块：学习更多特征，主要增加通道数
池化层：压缩空间尺寸，主要减小宽高
最终分类层：把抽象特征转换成 10 个类别分数
```

更一般地说，卷积是否会缩小特征图取决于 `kernel_size`、`padding` 和 `stride`。现代 CNN 中常见的设计是让普通卷积保持宽高不变，再用池化或 `stride=2` 的卷积负责下采样。

## 几个基础问题

### 训练是在调整参数，参数存储在哪里？

在 PyTorch 中，模型参数存储在模型对象内部。更具体地说，参数属于模型中的各个层，例如卷积层、线性层和 BatchNorm 层。

当代码中创建模型时：

- `model = Cifar10CNN().to(device)`

这个 `model` 对象内部就包含了所有可训练参数。卷积层中的卷积核权重、线性层中的权重和偏置、BatchNorm 中的缩放和平移参数，都会被 PyTorch 记录为可训练参数。

优化器通过 `model.parameters()` 拿到这些参数。训练时，前向传播会用当前参数生成预测结果；损失函数会衡量预测结果和真实标签之间的差距；反向传播会计算每个参数对应的梯度；最后优化器根据梯度更新参数。

因此，训练过程不是另外生成一份参数表，而是在不断修改 `model` 内部各层的参数。

### CNN 通常由哪些部分组成？

一个典型的卷积神经网络通常由多个卷积层和池化层组成。卷积层负责从局部区域提取特征，例如边缘、纹理、颜色组合和局部形状；激活函数负责引入非线性；池化层负责缩小特征图尺寸，减少计算量，并让模型对小范围平移更鲁棒。

常见结构可以理解为：

- 卷积层
- 激活函数
- 池化层

更现代的写法通常会加入 BatchNorm：

- 卷积层
- BatchNorm
- 激活函数
- 池化层

在这次作业中，模型的主体就是多个卷积功能块和池化层组成的特征提取器，最后接一个分类层输出 CIFAR-10 的 10 个类别。

### 一个 PyTorch 训练脚本通常是什么结构？

一个典型的 PyTorch 训练脚本通常包含以下几部分：

- 导入依赖和配置环境
- 定义超参数
- 准备训练集、测试集和 DataLoader
- 定义模型结构
- 定义损失函数、优化器和学习率调度器
- 执行训练循环
- 执行测试或验证循环
- 保存日志或模型

训练循环的核心顺序是：

- 把数据送入模型，执行前向传播
- 用损失函数计算预测误差
- 清空旧梯度
- 执行反向传播，计算新梯度
- 使用优化器更新参数

这次作业中的 `main` 函数负责把这些部分串起来，`train_one_epoch` 负责训练一个 epoch，`evaluate` 负责在测试集上评估模型。

### AdamW 优化器起什么作用？

优化器的作用是根据梯度更新模型参数。最基础的梯度下降可以理解为：参数沿着让 loss 下降的方向移动一小步。

Adam 会在普通梯度下降的基础上记录梯度的历史信息，为不同参数自适应地调整更新幅度。这样通常更容易训练，也不需要像普通 SGD 那样细致地手动调学习率。

AdamW 是 Adam 的改进版本，它更合理地处理权重衰减。权重衰减可以理解为一种正则化方法，会抑制参数变得过大，从而减轻过拟合。

在这次作业中，AdamW 用来更新 CNN 中的所有可训练参数。

### BatchNorm 是什么？

BatchNorm 是 Batch Normalization，中文通常译为批量归一化。它一般放在卷积层之后、激活函数之前。

它的作用是让中间特征的数值分布更加稳定。对 CNN 来说，`BatchNorm2d` 会按通道统计特征的均值和方差，并对特征进行归一化。同时，它也有自己的可训练参数，用来学习缩放和平移。

BatchNorm 的好处包括：

- 让训练更稳定
- 加快收敛
- 允许使用相对较大的学习率
- 带来一定正则化效果

需要注意的是，BatchNorm 在训练和测试时行为不同。训练时它会使用当前 batch 的统计量，并更新内部保存的 running mean 和 running var；测试时它会使用训练过程中保存下来的统计量。因此训练代码中需要正确切换 `model.train()` 和 `model.eval()`。
