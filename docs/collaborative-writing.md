# 协作写作流程

HomePage 的学习记录和项目文章采用两层协作：

1. **事实层由 Codex 先写。**
   包括背景、概念解释、过程、步骤、遇到的问题、证据、结果和可复用方法。
2. **个人理解层留给本人补充。**
   包括我的理解、感受、为什么在意、仍然模糊的地方、以及这件事和未来目标的关系。

这样做的目标是：让文章先有清晰结构和可靠事实，但不让 AI 代替本人完成反思。

## 默认结构

```md
---
title: ""
excerpt: ""
publishedAt: "YYYY-MM-DD"
updatedAt: "YYYY-MM-DD"
type: "study"
tags: []
topics: []
concepts: []
related: []
status: "draft"
---

## 背景

这里由 Codex 先写事实背景。

## 事实记录

这里由 Codex 整理概念、过程、步骤、问题、结果和证据。

## 我的理解

> 留给我补充：这件事我现在是怎么理解的？

## 我还没想明白的地方

> 留给我补充：哪些地方仍然模糊？

## 之后可能连接到

这里由 Codex 帮我列出相关概念、项目、笔记或后续问题。
```

## 发布规则

- Codex 可以先把文章放入 `content/drafts/`。
- 草稿中的“我的理解”“我还没想明白的地方”默认保留占位。
- 只有本人确认后，才迁移到正式 `content/blog/zh/`。
- 学习星图可以读取事实层和元数据，但个人理解层可以稍后补充。
