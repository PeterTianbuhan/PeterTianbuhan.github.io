## 任务目标

这次作业围绕 CIFAR-10 图像分类展开，要求使用 PyTorch 自定义卷积神经网络，在不使用预训练模型的前提下完成数据加载、模型定义、训练循环和测试评估，并在 100 个 epoch 内让测试集准确率超过 80%。

完成这次作业之后，我对一个典型 PyTorch 训练任务的结构有了更清晰的认识：先准备训练集和测试集，再定义模型结构、损失函数和优化器；训练时通过前向传播得到预测结果，通过损失函数衡量预测误差，再通过反向传播计算梯度，并由优化器更新模型参数。

这篇记录主要保存两类内容：一是这次 CNN 作业的实现和训练结果，二是围绕训练过程产生的几个基础问题，包括模型参数存在哪里、CNN 通常由哪些层组成、PyTorch 训练脚本的基本结构、AdamW 和 BatchNorm 分别起什么作用。

## 源码结构

本次作业的完整源码保存在：

- `D:\projects\ai_homework\homework_02_cifar10_cnn\cifar10_cnn.py`

这份代码可以按训练任务的流程分成几个部分来看。

首先是环境和基础配置。代码开头导入 PyTorch、数据加载、神经网络模块等依赖，并定义 CIFAR-10 数据集的均值和标准差。均值和标准差用于图像标准化，使输入数据的数值分布更稳定。

接着是设备选择和随机种子设置。`set_seed` 用来固定随机性，使实验更容易复现；`get_device` 用来选择训练设备，优先使用可用的 MLU 或 CUDA GPU，否则回退到 CPU。

然后是数据预处理和数据加载。`build_transforms` 定义训练集和测试集的图像处理流程：训练集使用随机裁剪、水平翻转、颜色扰动和随机擦除等数据增强方法，测试集只做张量转换和标准化。`build_dataloaders` 使用 `torchvision.datasets.CIFAR10` 读取数据集，并通过 `DataLoader` 构造训练和测试批次。

模型结构由 `ConvBlock` 和 `Cifar10CNN` 两部分组成。`ConvBlock` 是一个重复使用的卷积功能块，内部包含卷积层、BatchNorm 和 ReLU；`Cifar10CNN` 则把多个卷积功能块、池化层和最终分类层组合成完整的 CNN。输入图像经过卷积部分提取特征，最后通过线性层输出 10 个类别的预测结果。

训练逻辑主要在 `train_one_epoch` 中。每个 batch 会先经过前向传播得到预测结果，再用损失函数计算预测和真实标签之间的差距。随后调用 `loss.backward()` 计算梯度，并由优化器执行 `optimizer.step()` 更新模型参数。

测试逻辑在 `evaluate` 中。测试阶段会切换到 `model.eval()`，并使用 `torch.no_grad()` 关闭梯度计算，只评估模型在测试集上的 loss 和准确率。

最后，`main` 函数把这些部分串起来：解析命令行参数，构造数据集和模型，定义损失函数、优化器和学习率调度器，然后循环执行训练与测试，并把每个 epoch 的结果输出到日志中。

## 几个基础问题

### 训练是在调整参数，参数存储在哪里？

在 PyTorch 中，模型参数存储在模型对象内部。更具体地说，参数属于模型中的各个层，例如卷积层、线性层和 BatchNorm 层。

当代码中创建模型时：

- `model = Cifar10CNN().to(device)`

这个 `model` 对象内部就包含了所有可训练参数。卷积层中的卷积核权重、线性层中的权重和偏置、BatchNorm 中的缩放和平移参数，都会被 PyTorch 记录为可训练参数。

优化器通过 `model.parameters()` 拿到这些参数。训练时，前向传播会用当前参数生成预测结果；损失函数会衡量预测结果和真实标签之间的差距；反向传播会计算每个参数对应的梯度；最后优化器根据梯度更新参数。

因此，训练过程不是另外生成一份参数表，而是在不断修改 `model` 内部各层的参数。

### CNN 通常由哪些部分组成？

一个典型的卷积神经网络通常由多个卷积层和池化层组成。卷积层负责从局部区域提取特征，例如边缘、纹理、颜色组合和局部形状；激活函数负责引入非线性；池化层负责缩小特征图尺寸，减少计算量，并让模型对小范围平移更鲁棒。

常见结构可以理解为：

- 卷积层
- 激活函数
- 池化层

更现代的写法通常会加入 BatchNorm：

- 卷积层
- BatchNorm
- 激活函数
- 池化层

在这次作业中，模型的主体就是多个卷积功能块和池化层组成的特征提取器，最后接一个分类层输出 CIFAR-10 的 10 个类别。

### 一个 PyTorch 训练脚本通常是什么结构？

一个典型的 PyTorch 训练脚本通常包含以下几部分：

- 导入依赖和配置环境
- 定义超参数
- 准备训练集、测试集和 DataLoader
- 定义模型结构
- 定义损失函数、优化器和学习率调度器
- 执行训练循环
- 执行测试或验证循环
- 保存日志或模型

训练循环的核心顺序是：

- 把数据送入模型，执行前向传播
- 用损失函数计算预测误差
- 清空旧梯度
- 执行反向传播，计算新梯度
- 使用优化器更新参数

这次作业中的 `main` 函数负责把这些部分串起来，`train_one_epoch` 负责训练一个 epoch，`evaluate` 负责在测试集上评估模型。

### AdamW 优化器起什么作用？

优化器的作用是根据梯度更新模型参数。最基础的梯度下降可以理解为：参数沿着让 loss 下降的方向移动一小步。

Adam 会在普通梯度下降的基础上记录梯度的历史信息，为不同参数自适应地调整更新幅度。这样通常更容易训练，也不需要像普通 SGD 那样细致地手动调学习率。

AdamW 是 Adam 的改进版本，它更合理地处理权重衰减。权重衰减可以理解为一种正则化方法，会抑制参数变得过大，从而减轻过拟合。

在这次作业中，AdamW 用来更新 CNN 中的所有可训练参数。

### BatchNorm 是什么？

BatchNorm 是 Batch Normalization，中文通常译为批量归一化。它一般放在卷积层之后、激活函数之前。

它的作用是让中间特征的数值分布更加稳定。对 CNN 来说，`BatchNorm2d` 会按通道统计特征的均值和方差，并对特征进行归一化。同时，它也有自己的可训练参数，用来学习缩放和平移。

BatchNorm 的好处包括：

- 让训练更稳定
- 加快收敛
- 允许使用相对较大的学习率
- 带来一定正则化效果

需要注意的是，BatchNorm 在训练和测试时行为不同。训练时它会使用当前 batch 的统计量，并更新内部保存的 running mean 和 running var；测试时它会使用训练过程中保存下来的统计量。因此训练代码中需要正确切换 `model.train()` 和 `model.eval()`。

## 训练环境与结果记录

这次训练使用本机的 NVIDIA GPU 完成。GPU 本身负责执行大量矩阵运算和卷积运算，而 CUDA 可以理解为连接深度学习软件和 NVIDIA GPU 的计算平台。PyTorch 通过 CUDA 版本的 `torch` 把张量和模型参数放到 GPU 上，使前向传播、反向传播和参数更新都能在 GPU 上执行。

从使用者角度看，CUDA 的作用主要体现在两点：一是让 PyTorch 能识别并调用 NVIDIA GPU；二是提供适合深度学习计算的底层加速能力。代码中把模型和数据移动到 `cuda:0` 后，后续计算就会尽量在 GPU 上完成。
