## 中间区域必须承担页面职责

如果一个视觉模块删除后页面依然完全成立，它通常就只是装饰。个人主页首屏不应该靠很多小控件撑起来，而应该有一个明确的视觉重心。

这块中心视觉承担三件事：

- 给页面一个被记住的图形印象
- 在左右两列信息之间建立呼吸区
- 用有限动效提供“系统正在运行”的感觉

## 用 CSS 而不是重 JS

这里选择用 CSS 渐变、边框、粒子点和轻微呼吸动画，而不是整页画布动画，有两个原因。

第一，性能更稳定。第二，内容层级不会被动态效果抢走。

## 动效要服务于节奏

首页只需要三类动效：

- 进入时的 boot-in
- 视觉核本身的轻微漂移与脉冲
- 信息块 hover 时的响应

这就够了。更多动效通常只会把页面做得更吵。
